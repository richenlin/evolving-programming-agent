# Simple Mode - 快速修复

适用于问题修复、代码重构、代码审查。

## 状态文件

| 文件 | 用途 |
|------|------|
| `progress.txt` | 会话进度日志 |
| `feature_list.json` | 任务清单（复杂修复时） |

## 工作流程

### 1. 任务拆解（复杂修复必须）

**何时需要拆解**:
- 涉及多个文件
- 范围较大
- 根因不明确

**示例**:
```
❌ "修复登录失败"
✓ 1. 复现问题
  2. 检查前端请求
  3. 检查后端响应
  4. 定位失败点
  5. 修复代码
  6. 验证修复
```

### 2. 状态检查（每次会话必须）

```bash
cat progress.txt   # 了解历史
```

**检查后行动**:
- 从"下一步"继续
- 优先处理遗留问题

### 3. 修复循环

```
理解问题 → 复现 → 分析根因 → 制定方案 → 最小化修改 → 验证 → 状态更新 → commit → 进化检查
```

### 4. 状态更新（每步必须）

**progress.txt**:
```
## 本次完成
- [x] 修复内容

## 下一步
- 剩余工作

## 遇到的问题
- 问题 + 方案
```

## 验证方法

| 类型 | 方法 |
|------|------|
| 编译 | 运行构建 |
| 单测 | 运行测试 |
| 功能 | curl/手动 |
| 回归 | 确认无破坏 |

## 错误处理

1. 分析失败原因
2. 尝试不同方案（最多3次）
3. 连续失败: 回退 + 记录 + 报告用户

## 进化检查（循环结束后必须）

`@load workflows/evolution-check.md`
